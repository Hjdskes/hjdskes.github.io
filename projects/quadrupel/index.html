Quadrupel: an embedded software project to stabilize a drone

For the course IN4073 Embedded Real-Time Systems my project group and I had to program the software of an embedded control unit of a drone (dubbed the Quadrupel), to provide stabilization such that it can hover and fly with limited user control.

Our drone managed to take off, fly and land safely. Our response time was high enough for adequate stability, which we managed to do by:

  1. using a simple, custom message format on top of the COBS protocol to communicate between the base station (a PC) and the drone,
  2. sticking to a small and simple code base implementing a finite state machine,
  3. thoroughly reviewing each other’s pull requests.
